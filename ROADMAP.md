# Roadmap

Pilpul should stay small. The roadmap is not a growth plan; it is a stewardship plan.

## Now

Goal: make the current product trustworthy enough for the first real reading groups.

- Public repository preparation.
- Security and contribution docs.
- Screenshots and a clear README.
- Manual seed users from serious reading communities.
- Better smoke tests for auth, PDF import, request interests, and session room.
- Confirm production Resend and magic-link flow.
- Confirm Supabase storage policies and PDF access behavior.

## Next

Goal: help strangers reliably start and complete a text.

- Better first-session scheduling.
- Clearer queue/waiting expectations.
- Session recap flow.
- Pairing exit and report UX polish.
- Email notifications for accepted interests and direct invites.
- More robust notebook conflict handling.
- Better mobile PDF reader controls.

## Later

Goal: improve stewardship without turning Pilpul into a social network.

- Curated reading rounds.
- Optional public archive of anonymized post-completion notes.
- Maintainer dashboard improvements.
- True collaborative notebook transport, likely Yjs or another CRDT.
- Importers for specific public sources when they clearly help.
- PWA polish.
- UI translations.

## Not Planned

- Feeds.
- Likes.
- Follower counts.
- Public profile browsing.
- Streaks.
- Badges.
- Ad tracking.
- Recommendation algorithms designed to increase time in app.
- AI prompts that proactively interrupt the room.

## Open Questions

- Whether AGPL remains the best long-term license.
- Whether Jitsi remains sufficient for early voice/video use.
- Whether paid AI credits should exist, or whether bring-your-own-key is enough.
- Whether public notes should exist at all, even after completion.
- How much moderation tooling is needed before wider launch.
